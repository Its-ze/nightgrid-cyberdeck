Install / update

One page, all supported installers.

Direct release assets are available for manual downloads. Script installers update in place and keep the install path predictable.

Linux AppImage

Portable desktop build. The script installs or updates it in a writable user location and refreshes the launcher icon.

curl -fsSL https://its-ze.github.io/nightgrid-cyberdeck/install-linux.sh | bash

Linux deb

Debian/Ubuntu package with a NightGrid-owned lid-close policy so a mobile workstation can keep running closed.

sudo apt install ./NightGrid-Cyberdeck-Linux-x64.deb

Windows Setup

Downloads the latest setup executable and launches the updater. Close NightGrid first if the installer asks.

irm https://its-ze.github.io/nightgrid-cyberdeck/install-windows.ps1 | iex

Linux USB access may require the `dialout`, `uucp`, or `plugdev` group. Meshtastic CLI commands need exclusive access to the selected radio port.

USB setup notes

Deck surface

Built for repeated device work.

NightGrid keeps port scanning, serial sessions, command macros, and board-specific setup paths in one dense workspace.

Serial scanner

Role hints for Heltec V3, T-Deck, T-Dongle, ESP32, Flipper, Pico, GPS, CP210x, CH340, and Espressif USB serial.

Command Deck

Selected-device macros for help, status, mesh nodes, GPS push, T-Dongle status, ESP32 ping, and memory checks.

Marketplace

Tabbed packs for T-Deck, T-Dongle, ESP32, WLED, ESPHome, MicroPython, CircuitPython, ESP-NOW, LVGL, Matter, and more.

War Drive

Passive Meshtastic node sightings with GPS-stamped JSONL/CSV logs. Require GPS is on by default.

ESP32 Remote

Identify, heartbeat, Wi-Fi scan, I2C scan, GPIO write, and custom commands over MicroPython REPL or newline JSON Link.

Flasher lane

Open firmware pages, connect selected serial ports, reset boards, and place ESP32 targets into bootloader mode.

GPS panel

Probe NMEA receivers, cache fixes, and push a fix to the selected T-Dongle bridge when you choose.

Local safety

No cloud relay for serial logs, mesh output, GPS coordinates, or War Drive files. Review logs before publishing them.

T-Dongle bridge

Pair the dongle without guessing.

The wireless bridge panel separates USB-alive, Wi-Fi AP join, GUI reachability, profile save, pairing, and payload steps.

Probe GUI

Reads the CyberDeck Link attach heartbeat, buffers split JSON frames, and reports the AP and GUI URL.

{"cmd":"attachProbe"}

Start link

Writes the bridge URL, saves the T-Deck profile mirror, triggers the pair popup, and begins pairing.

{"cmd":"pairBegin","deckId":"itsz-tdeck"}

Finish link

Confirms the pair code, sends deck-ready, refreshes launchers, and opens the T-Dongle GUI when reachable.

CyberDeck-Link / http://192.168.4.1

Remove cleanly

Uninstall without losing local data by accident.

Uninstall scripts preserve app settings by default. Set the documented purge flag only when you intentionally want local settings removed.

Windows uninstall
